## Что важно сохранить

- Страница продаёт мастер-класс «Домашнее заваривание», не индивидуальный мастер-класс «Домашний бариста».
- Текущая цена: `5 000 ₽`.
- Длительность: `3 часа`.
- Формат: офлайн-группа до 8 человек.
- Запись работает через существующий проект `schedule-online/homebrew-schedule-sync`.
- Backend, cron, Google Sheets и `.env` находятся вне `barista-course` и не меняются этой страницей.
- В Tilda используются попапы `#consalt` и `#waiting_list`; не создавать видимые элементы с такими `id`.

## Что подготовлено

- Страница собрана как набор HTML-блоков для Tilda в порядке `00 → 05`.
- Блок расписания разделён из существующего `homebrew-block.html`: CSS в `02`, JS в `03`.

- Блок фото сделан горизонтальным слайдером на desktop и mobile: карточки идут в один ряд, листаются по горизонтали и открывают попап с фотоальбомом.
- `index.html` используется только для локального превью через VS Code Live Preview или локальный сервер.
- `tilda-block.html` оставлен как полный локальный архив и не обязан вставляться в Tilda целиком.

## Блок фото

- Источник данных: `https://api.barista-school.ru/api/homebrew-gallery.json`.
- Fallback: `https://159-194-202-120.sslip.io/api-fallback/api/homebrew-gallery.json`.
- Рендер находится в `05-page-scripts.html`, стили — в `04-bottom-content.html`.
- На desktop и mobile фотоальбомы идут в один горизонтальный ряд с `overflow-x` и `scroll-snap`.
- Внешняя белая рамка, декоративные градиенты по краям и вложенная карточка-контейнер удалены.
- Карточка фотоальбома сделана как `article role="button" tabindex="0"`, а не как `<button>` с блочными элементами внутри; это важно для стабильного рендера в Safari.
- Верхнее изображение карточки лежит в `.mbs-master-doma-page__gallery-img-wrap`, заполняет контейнер через `position: absolute; inset: 0; object-fit: cover`.
- Для Google Drive/LH3 картинок URL нормализуется: `=s800` для обложек карточек и `=s1600` для фото в попапе.
- По клику, Enter или Space открывается попап с большой фотографией, стрелками, счётчиком и лентой миниатюр.

## Как проверить

1. Открыть `master-doma/index.html` через VS Code Live Preview или локальный сервер.
2. Проверить desktop и mobile 390px.
3. Проверить, что на странице один H1.
4. Проверить, что `#consalt` и `#waiting_list` не используются как видимые `id`.
5. Проверить загрузку расписания из `api/homebrew.json`.
6. Проверить, что кнопки записи открывают запись.
7. Проверить блок тренеров: секция не должна содержать вложенную декоративную карточку вокруг hosted-виджета.
8. Проверить блок фото:
   - карточки идут в один горизонтальный ряд на desktop и mobile;
   - верхняя фотография каждой карточки начинается от верхней границы карточки без пустой полосы;
   - карточки открывают попап;
   - в попапе фотография видна целиком, работают стрелки и миниатюры.
9. Проверить FAQ, плавный скролл, meta/canonical и JSON-LD.
10. После ручной вставки в Tilda проверить страницу в редакторе и на опубликованном URL.
