# Tilda-блоки страницы `/capping`

Для продакшн-страницы в Tilda теперь вставляется один раз короткий loader:

```text
capping/tilda-loader.html
```

Он загружает актуальную страницу с сервера:

```text
https://api.barista-school.ru/api/capping-page.html
https://159-194-202-120.sslip.io/api-fallback/api/capping-page.html
```

Файлы `00 -> 05` остаются рабочим источником страницы. Их не нужно заново вручную вставлять в Tilda при каждой правке.

| Порядок | Файл | Что это | Зачем нужен |
|---|---|---|---|
| 00 | `00-seo-and-page-styles.html` | SEO и общий стиль страницы | Подключает Mulish, preconnect, static JSON-LD и CSS всех обычных блоков страницы |
| 01 | `01-top-content.html` | Верх страницы | Hero, «Для кого», «Как проходит», шапка онлайн-записи и контейнер `#mbs-capping-widget` |
| 02 | `02-schedule-widget-styles.html` | Стили расписания | Оформляет карточки каппингов, popup события, mobile/desktop состояния виджета |
| 03 | `03-schedule-widget-script.html` | Логика расписания | Загружает каппинги из API, включает fallback, кэш, popup события и проверку ссылок yClients |
| 04 | `04-bottom-content.html` | Низ страницы | «Зачем приходить», mount-point секции «Фото», FAQ, финальный CTA, контакты менеджера и URL публичных итогов каппинга |
| 05 | `05-page-scripts.html` | JS страницы | Meta/canonical, FAQ-аккордеон, плавный скролл, быстрая загрузка фотоальбомов через `gallery-index.json` + `gallery-albums/{id}.json`, публичные итоги каппинга, попап фотоальбома и font guard |

## Итоги каппинга в попапе фотоальбома

Источник итогов:

```text
https://api.barista-school.ru/api/public/cuppings/results
```

Логика на странице:

- фотоальбомы и итоги грузятся отдельно;
- связь идёт по одному ID: `album id = cupping_id`;
- если для карточки есть опубликованные итоги, попап открывает вкладку `Итоги`;
- если итогов нет, карточка и попап выглядят как обычный старый фотоальбом;
- если фото ещё не готовы, но итоги опубликованы, карточка остаётся на сайте и показывает текст `Итоги каппинга уже доступны, фото появятся позже`;
- в попапе итогов верх закреплён, низ с кнопкой `Посмотреть ближайшие даты` закреплён, список лотов скроллится внутри.

Публичные данные в этом блоке обезличены. На фронт приходят только лоты, количество оценок и общий средний балл. Имена, телефоны, email, userId, анкеты, комментарии, сырые ответы и детальные вкусовые критерии не используются и не должны добавляться в этот Tilda-блок.

Прямые ссылки на альбомы:

```text
https://baristaschool.ru/capping?album=<id>
```

Кнопка копирования на карточке сразу копирует такую ссылку. При открытии ссылки страница сама находит карточку и открывает её попап.

## Как обновлять страницу

1. Править файлы `00 -> 05`.
2. Запустить из корня проекта:
   ```bash
   node capping/build-capping.js
   ```
3. Проверить локальное превью `capping/index.html`.
4. Выложить `capping/hosted/capping-page.html` на сервер как `capping-page.html`.
5. После выкладки проверить `/capping` на desktop и mobile.

Build-скрипт пересобирает:

| Файл | Назначение |
|---|---|
| `capping/index.html` | Локальное превью с верхней preview-плашкой |
| `capping/tilda-block.html` | Полная архивная сборка страницы |
| `capping/hosted/capping-page.html` | HTML, который загружает Tilda-loader с сервера |

## Как вставлять в Tilda

1. Создать один HTML-блок.
2. Вставить содержимое `capping/tilda-loader.html`.
3. Сохранить страницу.
4. Дальше обычные правки страницы делать через `capping/tilda-blocks/` и серверный `capping-page.html`, без ручной замены блока в Tilda.
5. В секции «Фото» после блока «Зачем приходить» проверить загрузку реальных альбомов из `https://api.barista-school.ru/api/gallery-index.json`, горизонтальный scroll, открытие фотоальбома по клику/Enter/Space и закрытие попапа по крестику, overlay и Escape.
6. Для альбома `42832410` проверить вкладку `Итоги`: верхний блок и нижняя CTA-кнопка закреплены, описание и лоты скроллятся внутри, звёзды рейтинга выровнены.
7. Для старого альбома без итогов проверить, что пустой блок итогов не появляется.

## Что не трогать при вставке

- Не вставлять файлы `00 -> 05` в Tilda вручную, если на странице уже стоит loader.
- Не удалять комментарии в начале файлов, чтобы не путаться при повторной вставке.
- Не менять endpoint'ы расписания без отдельной задачи.
- Не менять `booking_url`-проверку yClients.
- Не добавлять персональные данные или детальные вкусовые критерии в публичный popup `/capping`.
