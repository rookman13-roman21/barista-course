# Tilda-блоки страницы `/capping`

Эти файлы нужно вставлять в Tilda отдельными HTML-блоками строго по порядку. Полный файл `../tilda-block.html` в Tilda целиком не вставлять: он слишком большой для одного HTML-блока.

| Порядок | Файл | Что это | Зачем нужен |
|---|---|---|---|
| 00 | `00-seo-and-page-styles.html` | SEO и общий стиль страницы | Подключает Mulish, preconnect, static JSON-LD и CSS всех обычных блоков страницы |
| 01 | `01-top-content.html` | Верх страницы | Hero, «Для кого», «Как проходит», шапка онлайн-записи и контейнер `#mbs-capping-widget` |
| 02 | `02-schedule-widget-styles.html` | Стили расписания | Оформляет карточки каппингов, popup события, mobile/desktop состояния виджета |
| 03 | `03-schedule-widget-script.html` | Логика расписания | Загружает каппинги из API, включает fallback, кэш, popup события и проверку ссылок yClients |
| 04 | `04-bottom-content.html` | Низ страницы | «Зачем приходить», FAQ, финальный CTA и контакты менеджера |
| 05 | `05-page-scripts.html` | JS страницы | Meta/canonical, FAQ-аккордеон, плавный скролл и font guard |

## Как вставлять в Tilda

1. Создать 6 HTML-блоков подряд.
2. Вставить файлы по порядку `00 -> 01 -> 02 -> 03 -> 04 -> 05`.
3. Не менять порядок: блок `03` зависит от контейнера из `01`, а блок `05` зависит от HTML из `04`.
4. После вставки проверить страницу на desktop и mobile.

## Что не трогать при вставке

- Не объединять файлы обратно в один HTML-блок.
- Не удалять комментарии в начале файлов, чтобы не путаться при повторной вставке.
- Не менять endpoint'ы расписания без отдельной задачи.
- Не менять `booking_url`-проверку yClients.
