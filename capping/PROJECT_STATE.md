## Итоги каппинга в карточках фотоальбомов

Как работает публикация:

1. Менеджер в Dashboard открывает `/photo-albums`, заходит в нужный фотоальбом каппинга и работает с блоком `Оценочный лист`.
2. Лоты и ответы участников хранятся в Dashboard, не в Tilda и не в файлах страницы `/capping`.
3. Чтобы итоги появились на сайте, менеджер ставит статус оценочного листа `published`.
4. Дополнительно сам фотоальбом должен быть активен для сайта. Это защита от ситуации, когда итоги опубликованы, но карточку мероприятия менеджер временно скрыл.
5. Страница `/capping` загружает обычные фотоальбомы и отдельно загружает `GET /api/public/cuppings/results`.
6. Если у карточки есть совпадение по `album id`, в попапе появляется вкладка `Итоги`. Если совпадения нет, старый фотоальбом выглядит как раньше.

Публичный JSON итогов не содержит персональных данных. На сайт уходят только:

- `id`, дата и компания каппинга;
- счётчики участников и заполненных листов;
- список лотов;
- количество оценок по каждому лоту;
- средний общий балл по каждому лоту.

На сайт не уходят:

- имена, телефоны, email и userId участников;
- сырые ответы и оценочные листы;
- анкеты участников;
- комментарии;
- детальные оценки по вкусовым критериям.

Текущая проверочная карточка:

```text
14 июня 2026
ККК Какой кофан крутой
album/cupping id: 42832410
```

## Публичный оценочный лист `/capping_list`

Отдельная общедоступная страница для оценки лотов на любых мероприятиях, без личного кабинета и без связи с опубликованными итогами на `/capping`.

Как устроено:

1. В Tilda на странице `https://baristaschool.ru/capping_list` вставляется loader `tilda_blocks_others/capping/public-cupping-score-sheet-loader.html`.
2. Loader загружает hosted HTML `https://api.barista-school.ru/api/public-cupping-score-sheet.html`.
3. Рабочий исходник фрагмента лежит в `tilda_blocks_others/capping/public-cupping-score-sheet.html`.
4. Данные не отправляются на сервер: всё хранится только в браузере участника в `localStorage` ключ `mbs-public-cupping-score-sheet-v2`.
5. После правки фрагмент нужно загрузить на сервер в `/var/www/html/api/public-cupping-score-sheet.html`.

Что есть в интерфейсе:

- анкета: имя, номер телефона, название каппинга и 4 вопроса про опыт/привычки;
- 6 лотов по умолчанию;
- настройка лотов: переименование, добавление и удаление с подтверждением;
- последний лот удалить нельзя;
- оценка по шкале 1-5 по критериям: сухой аромат, аромат мокрый / на сломе, кислотность, горечь, сладость, тело, послевкусие, общее впечатление;
- дескрипторы и заметки по лотам;
- итоговый экран с личным рейтингом;
- копирование результата в понятном текстовом виде для мессенджера;
- печать / сохранение в PDF через браузер;
- скачивание CSV для тех, кому нужна таблица;
- кнопка `Как заполнять` в шапке: открывает попап-инструкцию для новичка.

Что специально не делаем в публичном листе:

- не используем `cupping_id`;
- не требуем вход в личный кабинет;
- не используем Bearer token;
- не вызываем Dashboard endpoint'ы оценочных листов;
- не отправляем имя, телефон, анкету, оценки, дескрипторы и заметки на сервер;
- не связываем ответы с фотоальбомами и опубликованными итогами на `/capping`;
- не показываем технические форматы вроде JSON обычному пользователю.

## Tilda loader для вставки

В Tilda вставляется один HTML-блок:

```text
capping/tilda-loader.html
```

Loader:

- показывает короткий fallback про каппинг;
- загружает hosted HTML с primary URL;
- при ошибке пробует fallback URL;
- вставляет HTML в контейнер и запускает скрипты из загруженной страницы;
- не использует Tilda API и не меняет публикацию страницы.

## Как проверить

1. Открыть `barista-course/capping/index.html` через VS Code Live Preview. Превью собирается из файлов `capping/tilda-blocks/`.
2. Проверить desktop и мобильную ширину.
3. Убедиться, что кнопка «Выбрать дату» скроллит к расписанию.
4. Проверить, что карточки расписания и popup виджета работают так же, как в исходном виджете.
5. Проверить, что секция «Фото» после блока «Зачем приходить» быстро показывает skeleton/первые карточки, на desktop и mobile листается горизонтально, карточки открывают попап, стрелки и миниатюры листают фото, Escape и overlay закрывают попап.
6. Для карточки с опубликованными итогами проверить вкладку `Итоги`: верх и низ попапа закреплены, список лотов скроллится внутри, звёзды рейтинга выровнены, кнопка `Посмотреть ближайшие даты` всегда видна.
7. Проверить, что старый фотоальбом без опубликованных итогов визуально не изменился.
8. Проверить FAQ-аккордеон.
9. Проверить контакты финального CTA: телефон, Telegram, WhatsApp.
10. После вставки loader в Tilda проверить `#consalt` и `#waiting_list`, потому что эти действия остаются Tilda-попапами на странице.
