## Что подготовлено

- Собрана новая продающая страница услуги `/excu` одним самодостаточным HTML-блоком для Tilda.
- Онлайн-запись использует существующую механику `excursion-schedule-sync`: публичный JSON расписания и `booking_url` из события.
- Дизайн блока расписания переписан под планируемые экскурсии: крупные карточки без проблемных фото из API, дата, обжарочная компания, цена, длительность, места и кнопка записи.
- Hero-фото и фото блока «Программа» переведены на Tilda CDN (`static.tildacdn.com`).
- Блок «Программа» сделан визуальной сеткой из 4 карточек с фото в общей ширине страницы: контроль качества, зелёное зерно и отбор, оборудование и профиль, дегустация кофе. На mobile карточки листаются горизонтально через swipe/scroll-snap, чтобы блок не занимал много высоты.
- Frontend фильтрует прошедшие события, сортирует планируемые события по дате/времени и не зависит от конкретного `activity_id`.
- `booking_url` дополнительно проверяется на странице: разрешены только `https`-ссылки на домены yClients. Если ссылка некорректна, кнопка уводит в Telegram.
- Для якорных ссылок добавлен плавный скролл. Он срабатывает только если целевой элемент реально найден на странице, чтобы не ломать Tilda-якоря и попапы вроде `#consalt` / `#waiting_list`.
- Добавлен SEO-слой для страницы `/excu`: canonical/OG/meta через JS, статический JSON-LD `WebPage` / `Service` / `FAQPage` / `BreadcrumbList` и динамический JSON-LD `Event` по планируемым событиям из расписания.
- FAQ приведён к паттерну дизайн-системы: 2 колонки, sticky-заголовок, карточки вопросов, плавное раскрытие через `mbs-excu__faq-body`.
- Деплой, Tilda API, backend и yClients-логика не менялись.

## API расписания

Основной endpoint:

```text
https://api.barista-school.ru/api/excursions.json
```

Fallback endpoint:

```text
https://159-194-202-120.sslip.io/api-fallback/api/excursions.json
```

Кнопка «Записаться онлайн» берёт ссылку из поля `booking_url` конкретной экскурсии. Если мест нет, ссылки нет или ссылка не проходит frontend-проверку yClients-домена, показывается переход в Telegram.

На стороне страницы есть защита на случай проблем с доступом к серверу:

- сначала пробуется основной endpoint;
- затем fallback endpoint;
- если свежий запрос недоступен, но есть сохранённое расписание в `localStorage`, оно показывается из кэша;
- если данных нет совсем, показывается понятное состояние ошибки с кнопками «Попробовать ещё раз», «Лист ожидания» и «Написать в Telegram».

## Как проверить

1. Открыть `barista-course/excu/index.html` через VS Code Live Preview.
2. Проверить desktop и мобильную ширину.
3. Убедиться, что блок «Ближайшие экскурсии» подтягивает актуальные планируемые события и сортирует их по дате.
4. Нажать «Записаться онлайн» и проверить, что открывается актуальная ссылка записи.
5. Проверить, что некорректный `booking_url` не попадает в кнопку записи, а заменяется на Telegram.
6. Проверить плавный скролл по кнопкам «Смотреть ближайшие даты» и «К расписанию».
7. Проверить mobile-блок «Программа»: карточки должны листаться горизонтально, а не идти длинной колонкой.
8. Проверить пустое расписание/ошибку API через временную локальную подмену ответа только в копии файла, если понадобится.
9. После публикации проверить страницу в валидаторе Schema.org и Google Rich Results Test.

## Остаточные риски

- `index.html` использует iframe для локального превью, чтобы не дублировать большой Tilda-блок. В Tilda вставлять нужно именно `tilda-block.html`.
- Фото страницы завязаны на Tilda CDN; если файл удалят из Tilda, изображение нужно заменить на новый CDN URL.
- Сейчас «онлайн запись» означает переход по проверенному `booking_url` из JSON, а не встроенную форму с вводом контактов на странице.
