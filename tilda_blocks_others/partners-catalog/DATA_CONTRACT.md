## Источники Битрикс

- `UF_CRM_MBS_PARTNER_PUBLISH` / `XML_ID=MBS_PARTNER_PUBLISH_ON_SITE` — единственное новое поле, переключатель публикации.
- `UF_CRM_MBS_PARTNER_PUBLIC_DESCRIPTION` / `XML_ID=MBS_PARTNER_PUBLIC_DESCRIPTION` — публичная заметка / описание партнёра; показывается на карточке и в деталях.
- `UF_CRM_MBS_PARTNER_USEFUL_FOR` / `XML_ID=MBS_PARTNER_USEFUL_FOR` — чем полезен при открытии кофейни.
- `UF_CRM_1771701549130` — специальные условия для выпускников школы; показывается в подробной карточке, если поле заполнено и не содержит приватных данных.
- `TITLE` — название.
- `LOGO` — логотип компании; используется как источник, но в JSON попадает только безопасный публичный URL.
- `WEB` — сайт.
- `UF_CRM_1755378655628` — Telegram.
- `UF_CRM_1766177865141` — Instagram.
- `UF_CRM_1771954597298` — рабочая ссылка на публичную страницу партнёра; в JSON не выводится.
- `UF_CRM_1756816250945` — рабочий тип партнёрства; в JSON не выводится.

`LOGO` — основной источник логотипа. Если Битрикс отдаёт только закрытую ссылку портала, в `logo_url` ставится безопасный fallback: favicon сайта партнёра из `WEB`. Закрытые ссылки `baristaschool.bitrix24.ru` и относительные `/bitrix/components/...show_file.php` на фронт не уходят.

`summary` на карточке заполняется из поля `Публичная заметка / описание партнёра`. Автотекст по категории используется только как fallback, если публичная заметка пустая или не прошла security-check.

Категории рассчитывает backend-экспорт. Он не должен полагаться только на название компании. Для точных направлений сначала используется стандартное поле Битрикс `Сфера деятельности` (`INDUSTRY`): фильтрация воды, химия, лёд, напитки, специи/сиропы, какао, матча, посуда, мебель, форма, CRM, молоко. Для более общих сфер вроде «Продажа оборудования» и «Консалтинг» дополнительно учитываются явные признаки в названии/сайте. `INDUSTRY=TELECOM` на этом портале означает «Продажа оборудования» и попадает в категорию `equipment`, `INDUSTRY=UC_908ELH` означает «Посуда» и попадает в `tableware`, `INDUSTRY=UC_MBS_CHEM` означает «Химия для кофемашин» и попадает в `chemical`, `INDUSTRY=7` означает «Мебельное производство» и попадает в `bar`. Неизвестная сфера без явных признаков попадает в `other`, а не в `consumables`.

## Privacy

В публичном JSON запрещены:

- телефоны из CRM;
- обычные email из CRM-поля `EMAIL`;
- внутренние условия, скидки, инструкции менеджеру;
- Google Drive ссылки;
- Bitrix24 / Bitrix Disk ссылки;
- ответственные, авторы создания/изменения, служебные ID;
- ссылка на публичную страницу партнёра как отдельное поле;
- тип партнёрства;
- любые `UF_CRM_*` ключи напрямую;
- webhook URL, токены и значения из `.env`.

Новые описательные поля не используются. Если партнёру нужен email на сайте, это отдельное решение; обычный CRM-email не публикуется автоматически.

## Frontend fallback

`hosted-partners-catalog.html` и локальный `partners-catalog.html` должны:

- показывать демо-данные, если JSON недоступен при локальном открытии;
- в production loader-сценарии не показывать демо-данные при недоступном API, а показывать аккуратное состояние ошибки;
- не ломаться, если нет логотипа, описания, CTA или соцсетей;
- не показывать пустые секции в подробной карточке;
- фильтровать партнёров по категориям и поиску без backend-запросов;
- на `/all-partners` показывать сетку категорий;
- на `/all-partners?category=...` скрывать сетку категорий и сразу показывать компании выбранной категории;
- не сбрасывать категорию при поиске внутри категории; после очистки поля пользователь должен оставаться на той же странице категории;
- использовать словарь синонимов для частых запросов вроде кофемашина/эспрессо-машина, касса/POS/CRM/учёт, вода/фильтр/осмос, посуда/чашки/стаканы, сиропы/специи;
- в пустом результате показывать полезные действия: поискать тот же запрос во всех категориях, очистить поиск, вернуться к категориям, попробовать подсказки.
