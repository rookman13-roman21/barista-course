# Barista 3: База знаний

Проект: страница курса «Бариста любитель» для `https://baristaschool.ru/barista_3`.

Статус: локальная версия и Tilda-блоки подготовлены к публикации. Проект делался по аналогии с `home-barista`, без усложнения архитектуры и без новых зависимостей.

## Основные ссылки

- Публичная страница: `https://baristaschool.ru/barista_3`
- Tilda pageid: `73574173`
- Tilda alias: `barista_3`
- Запись для одного участника: `https://n1964448.yclients.com`
- Запись для двух участников: `https://n2230746.yclients.com`

## Структура

- `index.html` — локальная цельная версия страницы для проверки в браузере.
- `tilda-source.html` — исходный HTML, скачанный из Tilda. Это снимок источника, не основной файл для публикации новых блоков.
- `tilda-blocks/00-seo.html` — SEO-блок для вставки в настройки страницы Tilda / HTML inside head.
- `tilda-blocks/01-hero.html` — hero-блок, главный CTA и модальное окно выбора тарифа.
- `tilda-blocks/02-how.html` — блок с описанием формата обучения.
- `tilda-blocks/03-programme.html` — программа курса и CTA записи.
- `tilda-blocks/04-pricing.html` — стоимость и CTA записи.
- `tilda-blocks/05-trainers.html` — тренеры. Все упоминания бренда приведены к `MBS`.
- `tilda-blocks/06-faq.html` — частые вопросы.
- `tilda-blocks/07-cta.html` — финальный CTA-блок.
- `tilda-blocks/08-gallery.html` — галерея, заголовок: «Рабочее пространство».
- `vercel.json` — конфигурация для локального/внешнего статического деплоя, если понадобится.

## Что сделано

- Создана отдельная папка проекта `barista-3`.
- Страница адаптирована под курс «Бариста любитель»: 1 день, 3 часа, стоимость 10 000 ₽ за одного и 14 000 ₽ за двоих.
- Добавлен поп-ап выбора формата записи:
  - «Записаться одному» ведёт на `https://n1964448.yclients.com`.
  - «Записаться вдвоём» ведёт на `https://n2230746.yclients.com`.
- Кнопки «Записаться онлайн» в ключевых блоках открывают этот поп-ап.
- Для мобильной версии в логике поп-апа предусмотрена блокировка скролла страницы под модальным окном.
- Убран локальный якорь в последнем CTA-блоке, чтобы кнопка из hero не прокручивала страницу в самый низ.
- Все вхождения `МШБ` в рабочей папке `barista-3` заменены на `MBS`.
- Подготовлен SEO-блок `00-seo.html`:
  - canonical на `https://baristaschool.ru/barista_3`;
  - meta description;
  - Open Graph / Twitter Card;
  - JSON-LD: `Course`, `FAQPage`, `BreadcrumbList`, `LocalBusiness`.
- Заголовок галереи изменён на «Рабочее пространство».

## Важное при публикации в Tilda

- `00-seo.html` не вставлять как обычный визуальный блок. Его нужно переносить в HTML внутри `head` / настройки страницы Tilda.
- В настройках страницы Tilda желательно установить title:
  `Бариста любитель — курс за 3 часа | Московская школа бариста`
- Остальные файлы из `tilda-blocks` публиковать как отдельные HTML-блоки по порядку.
- Основной рабочий источник для публикации — папка `tilda-blocks`, а не `tilda-source.html`.

## Что проверить после публикации

- Открывается ли поп-ап по всем кнопкам «Записаться онлайн».
- Не скроллится ли фон под поп-апом на мобильных.
- Работают ли обе ссылки YClients:
  - один участник;
  - два участника.
- Не осталось ли старого текста `МШБ` в опубликованных блоках.
- Не срабатывает ли лишний скролл к последнему блоку при клике на CTA в hero.
- Корректно ли подтянулись SEO-теги и JSON-LD.
- На странице виден блок `08-gallery.html`, если он должен быть опубликован.

## Осторожно

- Не ломать класс `js-mbs-hb-booking-modal`: он привязан к открытию поп-апа выбора тарифа.
- Не удалять модальное окно из `01-hero.html`: остальные кнопки используют его через общий обработчик.
- Не менять ссылки YClients без проверки актуальных тарифов.
- Не использовать `tilda-source.html` как источник правок для новых блоков: там находится старый экспорт Tilda и много служебного кода.
- Если правится `index.html`, нужно отдельно переносить важные изменения в соответствующие файлы `tilda-blocks`.

## Если вернуться к проекту позже

1. Сначала проверить актуальность опубликованной страницы `https://baristaschool.ru/barista_3`.
2. Сравнить опубликованные блоки с файлами из `tilda-blocks`.
3. Если нужны правки, менять конкретный блок в `tilda-blocks`.
4. После правки прогнать поиск:
   `rg "МШБ|n1964448|n2230746|js-mbs-hb-booking-modal" barista-3`
5. После проверки локальной версии перенести обновлённый HTML-блок в Tilda.
