# PROJECT_STATE — open_cafe_app

Дата актуализации: 2026-06-24

## 1. Назначение

`open_cafe_app/` — лендинг платформы MBS* Coffee Menu для будущих владельцев кофеен.

Страница продаёт доступ к сервису, где можно до аренды, ремонта и закупок посчитать:
- стартовый бюджет;
- план продаж;
- дополнительные продажи;
- средний чек;
- food-cost;
- окупаемость;
- поставщиков;
- рецептуры;
- PDF/Excel-отчёты.

Публичный адрес страницы: `https://baristaschool.ru/open_cafe_app`.

## 2. Текущий артефакт

Рабочий файл: `open_cafe_app/index.html`.

На текущем этапе это один самодостаточный HTML-файл:
- CSS и JS внутри файла;
- без сборщика;
- без backend;
- без новых зависимостей, кроме Mulish и Phosphor Icons через CDN;
- классы с префиксом `mbs-openapp__`.

Страница пока не нарезана на Tilda-блоки и не деплоилась.

## 3. Оффер и CTA

Заголовок hero: «Платформа для открытия кофейни».

Короткий оффер: «Посчитайте бюджет, продажи, средний чек, окупаемость, поставщиков и отчёты до аренды и закупок.»

Цена: `18 900 ₽`.

В стоимость входит 1 час консультации с Романом Суслиным, основателем MBS*.

Покупка через Tilda-корзину:

```text
#order:Сервис открытия кофейни + консультация=18900
```

Заявка через Tilda-попап:

```text
#consalt
```

Локальный JS не должен перехватывать `#consalt`, иначе Tilda не откроет попап.

Основные требования:

## 5. Блок экранов сервиса

Блок показывает реальные экраны продукта. Основные карточки:
- «План продаж»;
- «Бюджет запуска»;
- «Рецепты»;
- «Фин.модель»;
- «Поставщики».

На desktop и mobile карточки идут в горизонтальной ленте.

По клику карточка открывает lightbox. В открытом виде:
- не переключаемся между основными карточками;
- переключаем только вложенные слайды выбранной карточки;
- доступны стрелки, клавиатура и свайп;
- счётчик слайда находится в зоне изображения и не перекрывает текст.

Дополнительные слайды:
- «Бюджет»: список оборудования, карточка оборудования;
- «Рецепты»: карточка напитка, режим редактирования рецепта, PDF техкарта;
- «Фин.модель»: калькулятор ФОТ, P&L, PDF-отчёт для инвестора или партнёра.

## 6. FAQ

FAQ работает как accordion: при открытии одного вопроса остальные закрываются.

Левый блок «Частые вопросы» использует sticky-паттерн как на `barista-courses`:
- `data-mbs-sticky-section`;
- `data-mbs-sticky-panel`;
- `data-mbs-sticky-offset`.

Не возвращать кастомную логику, которая двигает левый блок ниже правого списка FAQ.

## 7. Безопасность и приватность

В странице нет `.env`, токенов, API-ключей, логинов или паролей.

Скриншоты подключены только публичными ссылками Tilda CDN. Перед публикацией нужно ещё раз проверить, что изображения не содержат:
- приватных данных клиентов;
- внутренних токенов;
- закрытых админских URL;
- личных данных, которые нельзя показывать в рекламе продукта.

В браузере проверить:
- 390px;
- 768px;
- 1440px;
- отсутствие горизонтального скролла страницы;
- работу lightbox и вложенных слайдов;
- что `#consalt` не перехватывается локальным JS;
- что ссылка покупки сформирована как товар за `18 900 ₽`.
